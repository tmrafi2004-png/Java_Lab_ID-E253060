About Java: Java is a powerful and versatile programming language that allows you to “ write once, run anywhere ”. It is a high level, object-oriented, platform-independent programming language widely used for building web applications, mobile apps, enterprise software and more. It was designed to be simple, secure, portable and power.

History of Java: Java was developed by James Golling and his team at Sun 
Microsystems in 1991. 
Key Historical Timeline:  
	1991 - Project started as Green Project. 
	Originally called Oak. 
	1995 - Renamed to Java. 
	1996 - First official version (JDK 1.0) was released. 
	2010 - Sun Microsystems acquired by Oracle Corporation. 
	Since then, Oracle Corporation maintains and updates Java. 
Java became popular quickly because of internet growth in the 1990s. 

Feature of Java: 
	Platform Independents:  
	Java code is compiled into bytecode, which runs on the Java Virtual Machine (JVM).
	Java uses OOp principles: 
	Encapsulation 
	Inheritance 
	Polymorphism 
	Abstraction. 
	Robust:  
	Strong memory management. 
	Automatic garbage collection.
	Exception handling. 
	Simple:  
	Easy syntax.
	No complex features like pointers ( as in C++ ). 
	Source:  
	Built-in security manager
	Bytecode verification. 
	Multithreaded:  
	Supports multiple tasks running simultaneously. 

How Java Works: 
	Java source code (.java file) is written by the programmer.
	The Java compiler converts it into bytecode (.class file).
	The bytecode runs on the Java Virtual Machine (JVM). 
This JVM makes Java platform-independent.

Advantages of Java:
1.	Platform Independence (WORA): Code runs universally on diverse operating systems and hardware via the JVM, simplifying deployment.
2.	Object-Oriented: Supports core OOA principles (encapsulation, inheritance, and polymorphism), resulting in modular, reusable, and scalable code. 
3.	Security: Features like the bytecode verifier and sandbox environment ensure system protection against malicious code. 
4.	Automatic Memory Management: Garbage collection automates memory handling, preventing leaks and stabilizing applications. 
5.	Rich API & Ecosystem: Provides extensive standard libraries and is backed by a massive community, offering numerous third-party frameworks (e.g., Spring) and tools. 
6.	Multithreading Support: Native concurrency capabilities allow simultaneous execution, enhancing performance, especially for server applications. 
7.	Maturity and Community: Decades of industry presence ensure continuous updates, comprehensive documentation, and robust support.

Disadvantages of Java:
1.	Slower Performance: Java's execution via the JVM (bytecode interpretation/JIT compilation) can be slower than natively compiled languages (eg.. C++). 
2.	Higher Memory Consumption: The JVM requires significant memory overhead (for garbage collection and runtime), making Java applications resource-intensive.
3.	Complex Garbage Collection: Automatic garbage collection can cause occasional "GC pauses," and optimizing JVM memory tuning requires expert knowledge. 
4.	No Low-Level Access: As a high-level language, Java lacks direct hardware or memory manipulation (no pointers), limiting its use in systems programming or high-performance computing.
5.	Verbose Syntax: Java's often verbose code requires more lines for simple tasks compared to languages like Python or Kotlin, potentially slowing development and reducing readability.
